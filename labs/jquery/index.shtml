Write the JavaScript

In our previous labs we've concentrated on building and styling an HTML page. In this lab, we will instead concentrate on writing JavaScript. I've provided the basic HTML and CSS files for you, but you will add the JavaScript necessary to generate the address book entries, and resort them when the reader clicks on various buttons.

First, let's see what this page will look like when we are done:

If you look at the source for index.html, you will notice that the various address book entries are not actually in there. Instead, we will be generating the entries dynamically in JavaScript, based on a data structure that I have already populated for you.

The Sort By buttons at the top of the page are a Bootstrap button group, and we will write the JavaScript necessary to catch their click events, resort the address book by the respective property, and re-render the entries in the new sorted order, all without ever leaving the web page.

Let's build this up, step-by-step.

Review Data File and Template

Before we start coding, open the file 'employees.js' in your proejct's 'js' sub-folder so you can see what our data structure looks like. This script file creates one global object called Employees, which has one property called entries. This property contains an array of objects, each of which have the properties first (first name), last (last name), title (job title), dept (department), and pic (url to headshot). These are our address book entries.

Now open the index.html file in your project's root directory, and look at the <div class="template"> element near the bottom of the file. Our stylesheet has a rule for .template that sets display: none;, so the reader won't be able to see this, but we will use it in our code. In essence, we will merge the objects in Employees.entries with this template to produce the individual entries. As we build each new entry, we will append them to the <div class="address-book"> element, which will act as the container for all address book entries.

Write a Function to Sort the Entries by a Given Property

In order to do all this, we need to write a couple of functions, the first of which is a function that can sort the address book entries by any given property name. Open the file 'address-book.js' located in your project's 'js' sub-folder. Create a new function at the top of that file called sortEntries that accepts a single parameter named prop.

The body of your function should use the sort() method that is available on all JavaScript arrays (including Employees.entries). That function can sort arrays of primitive values on its own, but to sort an array of objects, you must supply a compare function as the first parameter, which the sort() method will call repeatedly passing two objects from the array. Your compare function should return a value less than 0 if the first object should be placed before the second, a value of 0 if they are equal, and a value greater than 0 if the first object should be placed after the second.

Since the caller of your function is passing the property to compare as a parameter, you'll want to use JavaScript's array-like syntax for looking up a property name using a variable. For example, to look up a property in an object obj corresponding to a name stored in a parameter called prop, the syntax would be obj[prop].

Since this is a little tricky, and since you've only had a couple classes on JavaScript so far, this is what your completed function should look like:


function sortEntries(prop) {
    Employees.entries.sort(function(a, b) {
        if (a[prop] < b[prop])
            return -1;
        else if (a[prop] == b[prop])
            return 0;
        else
            return 1;
        // return a[prop].localeCompare(b[prop]);
    });        
} //sortEntries()

Alternatively, since we know that all the properties are strings, we could just use the localeCompare() function instead of testing the individual cases:


function sortEntries(prop) {
    Employees.entries.sort(function(a, b) {
        return a[prop].localeCompare(b[prop]);
    });        
} //sortEntries()

Note that in a real system, you'd always want to include support for a secondary and even tertiary sort key. For example, sorting on only last name is fine for this small sample set because no two entries have the same last name. But in a real company, you'll have several people with the same last name, so you'd want to sort on first name within last name. If you're interested in adding this support, this is one of our optional steps.

Write a Function to Render the Entries

Now add another function to address-book.js called render(), which takes no parameters. This function will merge the objects in Employees.entries with the HTML in the <div class="template"></div> element.

The first thing this function should do is get a reference to the <div class="template"></div> element, and the <div class="address-book"></div> element. You'll need the former to clone(), and you'll append() the cloned template elements after you populate them to the latter. Use jQuery to select both of these elements by their class name, and assign them to variables.

Since this function will be called whenever the reader chooses to resort the entries, the first thing we need to do is clear out any HTML that might already be in the <div class="address-book"></div> element. Use jQuery's empty() method to do that.

Next, for each object in Employees.entries, we need to merge its properties with our template HTML. You can use a standard JavaScript for() loop for this, or you can try jQuery's $.each() function.

Inside your loop body, you'll first want to use jQuery's clone() method to clone the template element tree into a new element tree that you will populate. Assign the return value of clone() to a variable so you can work with the cloned elements.

The template HTML looks like this:


<div class="template person">
    <img class="pic" src="" alt=""/>
    <div class="details">
        <h2 class="name"><span class="first"></span> <span class="last"></span></h2>
        <p class="lead title"></p>
        <p>Department: <span class="dept"></span></p>
    </div>
</div>

You'll notice that for each property defined in an address book entry, there is a corresponding HTML element with a matching class name. For example, each address book entry has a first and last property, which correspond to the <span class="first"></span> and <span class="last"></span> element in the <h2> element. This will allow us to find and populate the HTML elements that correspond to our address book entry objects.

Use jQuery's find() method to find a descendant element of your cloned template with a class name that matches each of the properties in the entry objects: first, last, title, dept, and pic. For all properties except pic, use jQuery's html() method to populate the contents of the element. For the pic property, use jQuery's attr() method to set the src attribute of the <img> element to the value of the pic property.

After you've populated the cloned template elements, remove the 'template' class from the cloned template element using jQuery's removeClass() method. This will make the new entry visible on the page.

Finally, use jQuery's append() method to append the populated elements to the <div class="address-book"></div> element you selected earlier and stored in a variable.

Sort and Render the Entries on Load

Now that you have a function to sort the entries, and another function to render them, you next need to write the code to sort the entries by last name and render them when the page is fully loaded and ready for manipulation. The jQuery library has a handy mechanism for registering a function to be called after the page is loaded, so use that to call a new function that calls your sort() function passing 'last' as the property to sort upon, and then your render() function to render the entries.

If you've done everything correctly, you should be able to open the index.html page in your browser and see the various entries listed, sorted by last name.

Add Event Handlers for Sort Buttons

The last thing we need to do is respond appropriately when a reader clicks the "Sort By:" buttons at the top of the page. To do that, we need to register a click event handler for all the buttons, and within that function, resort by the appropriate property and re-render the entries.

First let's take a look at the sorting user interface (UI) markup:


<!-- sorting UI -->
<div class="sort-ui">
    <span class="sort-label">Sort By: </span>
    <div class="btn-group">
        <button type="button" class="btn btn-default active" data-sortby="last">Name</button>
        <button type="button" class="btn btn-default" data-sortby="title">Title</button>
        <button type="button" class="btn btn-default" data-sortby="dept">Department</button>
    </div>
</div>

Note that the <div> surrounding the button group has the class 'sort-ui', and the individual buttons have a style class of 'btn' amongst others. Use these two style classes to select all the button elements via jQuery, and register a single click event handler function for all of them at once.

In that click handler function, we need to know which property to sort upon, which is why I included an extra data-sortby attribute on each of the buttons. In our event handler, the this variable will be a refence to the button element that was clicked, so you can get access to this attribute value and use it to sort the entries by the appropriate column.
