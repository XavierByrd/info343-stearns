To finish, publish your site to your UW student/staff/faculty web hosting server. You can do this either via FTP (using FileZilla), or via SSH and Git. If you're not afraid of the command line, use the SSH/Git option. Follow the relevant steps below depending on your choice:

Publish via FTP

Follow these instructions written by UW IT to connect to your UW web hosting server using FileZilla. Once you have successfully connected, follow these steps:

  1. FileZilla shows you files on your local hard drive in the left box, and files on the remote server in the right box.
  2. In the right-side file listing box, double click the 'student_html' folder if you have one, otherwise double click the 'public_html' folder.
  3. If you don't already have a folder there named 'info343', right-click and choose 'Create Directory' to create a new directory. Name it 'info343' (all lower case--remember that URLs are case-sensitive).
  4. Double click the 'info343' directory.
  5. Right-click again and choose 'Create Directory'. Name this new directory 'labN', replacing N with this lab number (e.g., lab1, lab2, lab3, etc.). Keep the 'lab' part all lower case.
  6. Double-click the 'labN' directory you just created.
  7. Now in the left-side file-listing box, find your local repository on your hard drive.
  8. Select all your website files (don't include the .git folder) and drag-drop them from the left box to the right box.

Publish via SSH/Git

If you're not afraid of the command line, you can publish your site, and keep it updated quite easily by using SSH and Git. Follow these steps:

  1. Go to your repository for this lab on the GitHub web site. On your repository home page, there should be a box on the right entitled 'HTTPS clone URL'. Click the copy to clipboard button, or select the URL and copy it to your clipboard.
  2. If you are on Mac OS X or Unix, open a Terminal window and SSH to the web server by entering the command ssh your-netid@dante.u.washington.edu, replacing your-netid with your UW Net ID. Enter your UW NetID password at the prompt.
  3. If you're on one of the Windows lab machines, go to the home screen and click on the icon for 'Student Terminal (Dante)' in the 'Remote Access, FTP' section (far right). Enter your UW NetID and password when prompted.
  4. If you are on a non-lab Windows machine (say at home), download and install Tera Term and configure it to connect to your web hosting server.
  5. Once connected, type ls and hit return to list the files and sub-directories in your home directory. If you have a directory named 'student_html' you'll want to use that. If not, use the 'public_html' directory.
  6. Change to the student_html directory if you have one, otherwise change to the public_html directory (cd student_html, or cd public_html).
  7. List the files again (ls) to see what you already have there. If you don't already have a directory named 'info343', create one: mkdir info343
  8. Change to the info343 directory (cd info343).
  9. Now use Git to clone your repository into a new sub-directory. To do that, Execute this command: git clone your-clone-url, replacing your-clone-url with the URL you copied on the GitHub site. If you're using Tera Term on a Windows machine, use the right mouse button to paste the URL you copied from GitHub. If you're on a Mac, use the normal paste keyboard command (Cmd+v). By default, git clone will create a directory with the same name as your GitHub repository, but you can also override that by adding a different name after your clone URL (e.g., git clone https://.../xyz.git mylab.
  10. You should see a number of messages fly by, with the final one being '100% ... done.' If you see an error message, read it to figure out what went wrong.
  11. Change to the directory just created and list the files. You should see the files from your GitHub repository.
  12. If you make any future changes to your files, commit them, and push them your GitHub repository, you can then quickly update your site on the UW web hosting server by executing git pull from within the directory created when you cloned. Git will pull down any changes made since the last time you cloned/pulled, and apply them.

If everything went well, you should now be able to see your web site in a browser using the URL http://[students|staff|faculty].washington.edu/your-net-id/info343/lab-dir/. Use 'students', 'staff' or 'faculty' depending on the affiliation you had when you activated your web publishing service. Replace your-net-id with your UW net id, and replace lab-dir with the name of the directory you created to hold these lab files.
