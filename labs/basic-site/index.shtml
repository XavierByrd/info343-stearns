Build and Style the HTML Pages

I include sample screen shots below for the various pages and common content, and you should strive to get your versions looking as close to these screen shots as possible. They don't need to match exactly but try to get them as close as you can. In a professional project, you'll often get screen shots like these from a graphic designer, and your job as a web developer is to turn those into real live web pages.

Now that you have a local clone of your repository, you'll next build and style the HTML pages for the site. You'll build two pages for the "Widgets-R-Us" corporate web site: a home page (index.html), and an about page (about.html).

If you finish early and want an extra challenge, try adding a products page as well, which is detailed in the optional steps below.

Each page you build should link to your main stylesheet ('css/styles.css'), which I have started for you. This stylesheet has three @include directives at the top. The first two include the Google web fonts 'Boogaloo' and 'Open Sans'. We'll use Boogaloo only for the corporation name (Widgets-R-Us), and we'll use Open Sans for all the rest of the text on the site.

The other @include directive adds another stylesheet named 'reset.css', which is quite handy. Different browsers have different defaults for things like margins, paddings, borders, and alignments, so this CSS file will reset all of those, giving you a clean slate upon which to apply your own styles.

Each of the site pages will have the same header and footer content, so we'll put those into separate files and include them in each page using server-side includes.

The Home Page

Create a new file in your text editor and save it in the root directory of your repository as 'index.html'. This will contain the content for the following home page:

Start by including the HTML5 doctype declaration, and then create the basic <html>, <head>, and <body> elements that all pages need to have.

In the <head> section, add all the necessary meta-data elements. You'll need at least these:


<!-- tells the browser this file was saved in the UTF-8 character encoding -->
<meta charset="UTF-8">

<!-- tells the browser to get style information from styles.css in our 'css' sub-directory -->
<link rel="stylesheet" href="css/styles.css">

<!-- title for the page -->
<title>Widgets-R-Us</title>

<!-- REPLACE 'your uw net id' with your UW net id -->
<meta name="author" content="your uw net id">

You'll notice in the screen shot that the background of the page is a light-gray patterned texture. That texture already in your 'img' sub-directory in the file 'texture-light.png'. Use this as the background-image for the body. Remember that when specifying a URL in a stylesheet, the URL is a relative path starting from the location of the css file. Since our CSS file is in the 'css' sub-directory, and the texture is in a peer 'img' sub-directory, the relative path first needs to go up one level to the root directory (..) and then back down into the 'img' sub-directory. Here's what your body CSS rule should look like:


body {
    font-family: 'Open Sans', sans-serif;
    font-size: 18px;
    background-image: url(../img/texture-light.png);
    background-repeat: repeat;
    padding: 10px;
}

The main page content goes in a white box that should be sized to 80% of the body, and horizontally centered. It should also have a very subtle box shadow, similar to what is shown in the screen shot. We covered how to do this in class, but if you've forgotten, look at the page source of the box styling example page.

You've already created the common header and footer files, so use server-side includes to include those into the page at the appropriate place. The syntax looks like this:


<!--#include virtual="...relative url to include file..." -->

The picture of the factory is already in your 'img' directory in the file named 'factory.jpg'. The Free Shipping picture is also in your 'img' directory in the file named 'free-shipping.jpg'. These images should float to the left and right respectively, so that the text flows around them.

The rest of the text is just lorem ipsum text, which you can generate in Sublime (type 'lorem' and hit tab), or via a lorem ipsum generator site.

Publish via FTP

Follow these instructions written by UW IT to connect to your UW web hosting server using FileZilla. Once you have successfully connected, follow these steps:

  • FileZilla shows you files on your local hard drive in the left box, and files on the remote server in the right box.
  • In the right-side file listing box, double click the 'student_html' folder if you have one, otherwise double click the 'public_html' folder.
  • If you don't already have a folder there named 'info343', right-click and choose 'Create Directory' to create a new directory. Name it 'info343' (all lower case).
  • Double click the 'info343' directory.
  • Right-click again and choose 'Create Directory'. Name this new directory 'lab2' (all lower case).
  • Double-click the 'lab2' directory you just created.
  • Now in the left-side file-listing box, find your local repository on your hard drive.
  • Select all your website files (don't include the .git folder) and drag-drop them from the left box to the right box.

If everything went well, you should now be able to see your web site in a browser using the URL http://[students|staff|faculty].washington.edu/your-net-id/info343/lab2/. Use 'students', 'staff' or 'faculty' depending on the affiliation you had when you activated your web publishing service, and replace your-net-id with your UW net id.

Publish via SSH/Git

If you're comfortable at the command line, you can publish your site, and keep it updated quite easily by using SSH and Git. Follow these steps:

  • Go to your repository on the GitHub web site. On your repository home page, there should be a box on the right entitled 'HTTPS clone URL'. Click the copy to clipboard button, or select the URL and copy it to your clipboard.
  • If you are on Mac OS X or Unix, open a Terminal window and SSH to the web server by entering the command ssh your-netid@dante.u.washington.edu, replacing your-netid with your UW Net ID. Enter your UW NetID password at the prompt.
  • If you're on one of the Windows lab machines, go to the home screen and click on the icon for 'Student Terminal (Dante)' in the 'Remote Access, FTP' section (far right). Enter your UW NetID and password when prompted.
  • Once connected, type ls and hit return to list the files and sub-directories in your home directory. If you have a directory named 'student_html' you'll want to use that. If not, use the 'public_html' directory.
  • Change to the student_html directory if you have one, otherwise change to the public_html directory (cd student_html, or cd public_html).
  • List the files again (ls) to see what you already have there. If you don't already have a directory named 'info343', create one: mkdir info343
  • Change to the info343 directory (cd info343).
  • Now use Git to clone your repository into a new sub-directory. To do that, Execute this command: git clone your-clone-url, replacing your-clone-url with the URL you copied on the GitHub site. By default, git clone will create a directory with the same name as your GitHub repository, which in this case should be 'lab2'.
  • You should see a number of messages fly by, with the final one being 'Resolving deltas: 100% ... done.'
  • Change to the directory just created (cd lab2) and list the files. You should see your web site files.
  • If you make any future changes to your files, commit them, and push to your GitHub repository, you can then quickly update your site on the UW web hosting server by executing git pull from within this 'lab2' directory. Git will pull down any changes made since the last time you cloned/pulled, and apply them.
