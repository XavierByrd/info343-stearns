Homework Assignments

Requirements and supporting materials

Congratulations! You've just been hired by the hottest new pizzeria, Dawg Pizza, to build their web site. You will be working with the owner and founder, Tony Pizzarelli, to build a site that both he and his target customers will love.

You will build this site in four phases, adding complexity and features each time. We will assign these phases the version numbers 1.0, 2.0, 3.0, and 4.0. The requirements and due-dates for each version are listed below, and you'll submit your completed versions via the respective Canvas assignment.

For each of these versions, Tony will supply you with his list of must-have and optional requirements, plus some images and text copy you should include in the pages. If you want to ask clarifying questions, feel free to post them to our "Questions for Pizzeria Owner" discussion board in Canvas. The design of the pages will be up to you, however, so be creative and design something that both the owner and his target customers will find engaging, intuitive, and accessible. There is no one right way to build these sites, but we will examine your source to make sure you've used the appropriate HTML, CSS and JavaScript constructs for your particular solutions.

In order to get a full 10 points for the "completeness" portion of your grade, you must meet all the "must-have" requirements and at least one of the optional requirements. If you only do the must-have requirements, you'll get only 9 points on that portion of your grade. See the rubric displayed below each homework assignment in Canvas.

When you complete a version of your site, publish it to a new folder on your iSchool student web server and submit the URL via the Canvas assignment. If you don't link to the new folder from anywhere, it will remain an "island" on the web and no one else besides your instructor and TA will be able to find it. If you want to publish a link to the folder on the Internet, feel free to do so, but there is no requirement to do so.

Although you are not explicitly required to use a version control system while developing the various versions of this site, I would strongly recommend that you do so. A version control system like Git/GitHub will not only help you manage changes and recover from serious mistakes, but also keep your code backed-up in an offsite location in case your computer is lost, stolen, or broken. This is also a great opportunity to practice using these tools so that you are ready to jump into a professional web development team.

These are individual homework assignments, and this should be entirely your own work. Do not work on these sites together, and do not share code with each other. You may use open-source CSS frameworks and JavaScript libraries developed by others, but your own code must be your own code. If you get stuck on a technical question, you may ask for help on our "Technical Help" Canvas discussion board, but don't expect anyone to reply at 10pm on the night before the assignment is due.

Version 1.0 Due by Oct 8

Tony is working hard to get his pizzeria ready for their grand opening on October 8th, and he wants some kind of web presence by then. Below are his specific instructions. Remember that he isn't a web developer, so you'll need to interpret what he says and use the appropriate markup and styles. Color choices and page layout are entirely up to you unless otherwise stated. You can also include additional images, background textures, or icons wherever you think they'd be helpful.

If you draw a blank on how it should look, check out some other pizzeria web sites and see what they did.

Version 2.0 Due by Oct 22

That quick site you built is the doing the job, but now that Dawg Pizza is open for business, Tony wants you to refresh the site with some better page layout and visual effects. He also wants to make the site accessible to people with disabilities (they love pizza too), and that it looks good on smaller-screen devices like mobile phones and tablets. Lastly, he really wants the page to include the right meta-data so that Google and other search engines understand that this site is for a restaurant, and display it in search results with extra helpful information.

For this version, Tony has requested a few specific page layouts, but other design elements like fonts, text styling, box styling, transforms, transitions, animations, etc. are up to you. Remember that your goal is to build a site that his target customers (university students) would love. So be creative and utilize those skills you learned and practiced over the last few weeks.

If you want to use a CSS framework (e.g., Bootstrap or Foundation) for this version, feel free to do so, but remember to follow their licensing requirements.

Here is the message from the owner:

Hey, that first site was great, but I want you to do a couple of things to make it more engaging and useful. Here are my must-haves:

  • On the home and about pages, I want our logo to be off to the left with the rest of the text next to it on the right.
  • On the menu page, I'd like a two-column layout for the various pizzas, with the meat ones on the left and the veggie ones on the right. The drinks and desserts should be below that in just one column.
  • I want the whole site to look great on a mobile phone or tablet. The text needs to adjust size, pictures need to scale, etc. Also on pages like the menu, it needs to flip to a single-column layout when viewed on a phone screen so that customers don't have to scroll to the right to see the veggie pies.
  • I want the whole site to be accessible to people with disabilities.
  • I want Google and other search engines to know that this site is for a restaurant, so that they'll show extra helpful info in the search results (map, logo, phone number, hours, cuisine, link to menu, etc.)

If you get time, I'd also love one or more of these:

  • Some cool visual effects when people hover their mouse over various things.
  • It would be cool if the footer on all the pages was fixed on the bottom so that it stayed in place as you scrolled down the page.
  • Add a "Follow Us on Twitter" button that would automatically add the customer as a follower (provided the customer has a Twitter account of course).

Version 3.0 Due by Nov 5

Your redesigned site really helped attract customers to Tony's restaurant! He's now getting so busy that he wants to enable customers to place their delivery orders via the web site instead of having to call. For this version, you'll add some forms and scripting to make that possible.

Since we don't get into building databases and server-side processing in this course, you'll need to configure your page to post to a page I've already built for you on the server. See the instructions below the owner's message for details.

Here are your instructions from Tony:

Hey I need another feature added to the site. I want our customers to be able to submit a new delivery order right from the site. Here's how I want it to work:

  • I want a new page named "Order". It should be part of the main site navigation along with the existing pages.
  • That page should start with some text saying that we only deliver within the Seattle city limits, and only from noon to 11:00pm. We accept cash or credit cards on delivery.
  • It should then let the customer specify an order. The customer can order up to 5 pizzas, but each one has to be one of our standard options on the menu (no topping subsitutions).
  • For each pizza ordered, the customer needs to choose one of our standard sizes.
  • In addition to pizzas, the customer should also be able to order up to 10 of our soft drinks (no alcohol for delivery).
  • The customer should also be able to order up to 10 of our desserts.
  • The customer must choose at least one pizza to place an order (no orders for just drinks).
  • Below the food and drink choices, you should show a sub-total that dynamically updates based on what the customer chooses (see the menu for prices). Under that, you should show the standard delivery fee of $5. Below that you should show the tax amount, which is 9.5% of the sub-total + delivery. At the bottom, display the total charge (sub-total + delivery + tax).
  • Below the food and drink selection, there should be a second set of fields that let the customer specify the delivery address and phone number. The delivery address is required and must be within the city of Seattle.
  • Below the delivery address, the customer should be able to specify whether the pizza should be delivered as soon as possible, or at a specific time later in the day or evening. If the customer specifies a time, it has to be at least 30 minutes from the current time, no earlier than noon, and no later than 11:00pm that same day.
  • Below the delivery time, there should be a field for them to enter any special delivery instructions they might need (like, "come around back," or "please knock instead of ringing doorbell.") Limit this to only 140 characters though, since that's all we can print on our delivery tickets. As the customer types, show how many characters are left, and stop them when they get to 140.
  • At the bottom should be a button to place the order, and another to cancel and return to the home page [see note above about what submit URL to use for your form].

If you have the time, add one or more of the following:

  • Disable "place order" button until the customer has chosen at least one pizza, and provided a delivery address. Also disable the button after it is clicked so that the cusotmer doesn't click it multiple times and submit multiple orders.
  • Store the customer's address locally and auto-populate it on their next order.
  • Store the customer's entire order locally, and offer a UI element that the customer can click to place the same order as last time.

Since this class is limited to the client side, I've supplied a server-side page for you that can receive the data posted by your form, and echo back to the customer an order confirmation and receipt. The URL for this page is https://courses.washington.edu/info343/stearns/homework/placeorder.php.

This page accepts the following form field names:

Field Name Meaning
pizza-name-N The name of a pizza to order, exactly as it appears on the menu (e.g., 'Classic Pepperoni'). Replace N with the ordinal number, 0 through 4. For example, the first pizza ordered should be specified in the form field named 'pizza-name-0'. Specify at least one, and up to five pizzas.
pizza-size-N The size of the pizza, specified in inches. Valid values are 12, 14, 17, and 25. Replace N with the ordinal number, 0 through 4. For example, the size of the first pizza ordered should be specified in the form field named 'pizza-size-0'. The number of these fields should match the number of pizza-name-N fields you send.
drink-name-N The name of a soft drink to order, exactly as it appears on the menu (e.g., 'Coke'). Replace N with the ordinal number, 0 through 9. For example, the first soft drink ordered should be specified in the form field named 'drink-name-0'. These fields are optional (no drinks), but you can specify up to 10 of them.
dessert-name-N The name of a dessert to order, exactly as it appears on the menu (e.g., 'Gelato'). Replace N with the ordinal number, 0 through 9. For example, the first dessert ordered should be specified in the form field named 'dessert-name-0'. These fields are optional (no desserts), but you can specify up to 10 of them.
dessert-flavor-N The flavor of a dessert to order, exactly as it appears on the menu (e.g., 'Chocolate'). Replace N with the ordinal number, 0 through 9. For example, the flavor for the first dessert ordered should be specified in the form field named 'dessert-name-0'. The number of these fields should match the number of dessert-name-N fields you send.
