Homework Assignments

Requirements and supporting materials

Congratulations! You've just been hired by the hottest new pizzeria, Dawg Pizza, to build their web site. You will be working with the owner and founder, Michael Chen, to build a site that will both he and his target customers will love.

You will build this site in four phases, adding new features and capabilities each time. We will assign these phases the version numbers 1.0, 2.0, 3.0, and 4.0. The requirements and due-dates for each version are listed below, and you'll submit your completed versions via the respective Canvas assignment.

For each of these versions, Michael will supply you with his list of must-have and optional requirements, plus some images and text copy you should include in the pages. If you want to ask him to clarify any of these requirements, feel free to post a question to our "Questions for Pizzeria Owner" discussion board in Canvas. The design of the pages will be up to you, however, so be creative and design something that both the owner and his target customers will find engaging, intuitive, and accessible. There is no one right way to build these sites, but we will examine your source to make sure you've used the appropriate HTML, CSS and JavaScript given your solutions.

In order to get a full 10 points for the "completeness" portion of your grade, you must meet all the "must-have" requirements and at least one of the optional requirements. If you only do the must-have requirements, you'll get only 9 points on that portion of your grade. See the rubric displayed below each homework assignment in Canvas.

When you complete a version of your site, publish it to your iSchool student web server and submit the URL via the Canvas assignment.

Although you are not explicitly required to use a version control system while developing this site, I would strongly recommend that you do so. A version control system like Git/GitHub will not only help you manage changes and recover from serious mistakes, but also keep your code backed-up in an offsite location. This is also a great opportunity to practice using these tools so that you are ready to jump into a professional web development team.

These are individual homework assignments, and this should be entirely your own work. Do not work on these sites together, and do not share code with each other. You may use open-source CSS frameworks and JavaScript libraries, but your own code must be your own code.
