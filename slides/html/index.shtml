A very brief history of HTML

In the late 1980s, Tim Berners-Lee was working at CERN and looking for a way to keep track of all the people there, what projects they were working on, and how they all related.

Hypertext seemed like a great idea, but existing tools were too static and centralized.

What if pages were simple text files on on multiple hosts, with an easy way to link between them?

"Suppose all the information stored on computers everywhere were linked.... Suppose I could program my computer to create a space in which everything could be linked to anything. All the bits of information in every computer at CERN, and on the planet, would be available to me and to anyone else. There would be a single global information space."
"Once a bit of information in that space was labeled with an address, I could tell my computer to get it. By being able to reference anything with equal ease, a computer could represent associations between things that might seem unrelated but somehow did, in fact, share a relationship. A web of information would form."

Time Berners-Lee, Weaving the Web, p 4

Tim Berners-Lee with an early web browser (CERN 1991)

First web browser was an augmented version of the NeXT word processor; browsing and editing in the same tool.

Simple markup language (HTML) was intended to encode meaning into otherwise plain text.

Multipe types of "agents" could display HTML in context-appropriate ways (graphical, console text, print).

Semantics ≠ Presentation

But as the web caught on, people started using HTML elements to achieve particular visual effects, even if the semantics were all wrong.

Presentaiton-oriented tags were added (<b>, <i>, <font>).

Semantics and presentation started to meld...

...and HTML descended into CHAOS...

CSS came along in the late-1990s to re-separate semantics and presentation (hooray!).

Meanwhile, the eXtensible Markup Language (XML) was gaining popularity, and some thought it was a better solution for semantic markup than HTML.

Thus was born the beast...XHTML...

...great idea, but too many arbitrary rules....

HTML

<!doctype html>
<html>
    <head> ... <head>
    <body>
        <p>case-insensitive, end tags sometimes optional,
        marker attrs OK
        <img src="mycat.jpg">
        <button disabled>Click Me!</button>
    </body>
</html>

XHTML

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
"http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html xmlns="http://www.w3.org/1999/xhtml">
    <head> ... </head>
    <body>
        <p>case-sensitive, end tags or self-close required,
        attrs must have a value</p>
        <img src="mycat.jpg" />
        <button disabled="disabled">Click Me!</button>
    </body>
</html>

XHTML never really caught on, mostly because it was such a pain to write!

An alternative plan was hatched by a group of renegades, which soon became known as "HTML 5."

Intent was to clean up HTML, keep it simple, and return to a focus on semantics.
