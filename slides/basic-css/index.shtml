HTML Clarifications

  • Which elements need to be in a paragraph and which don't?

  • Why do I need an alt="" attribute on an <img>?

HTML Content Types

  • Paragraphs (p, pre, blockquote)
  • Other Block elements (ol, ul, dl, figure, table, div, h1-6) are peers to paragraphs, but have more specialized content
  • Phrasing elements (em, strong, sub, sup, q, ...) add meaning to segments of text within paragraphs or other elements
  • Embedded content (img) adds content in-line, meaning no explicit line break following
  • Hyperlinks (a) add linking to segments of text, images, or anything you choose to put inside
  • Sectioning content (article, nav, aside, header, footer, section) wraps around one or more paragraphs or block elements to define major sections of the page

Why <dl> for a Menu?

Goal is to use the most semantically-correct elements; that is, the elements that are closest in meaning to the content

A two-level <ul> might give you a hierarchical look, but it means a list of independent items, not a set of related names and descriptions.

A <pre> might let you control spacing in the presentation, but it only means a general "pre-formatted paragraph," not a set of related names and descriptions.

A <dl> means a set of related terms and descriptions, which is closest in meaning to a menu.

Properly Nested Lists

A sub-list always goes inside the <li> to which it belongs


<ul>
    <li>Item 1
        <ul>
            <li>Sub-item 1.1</li>
            <li>Sub-item 1.2</li>
        </ul>
    </li>
    <li>Item 2
        <ul>
            <li>Sub-item 2.1</li>
            <li>Sub-item 2.2</li>
        </ul>
    </li>
</ul>

Why Does <img> Need alt=""?

Because blind people use the web too! Screen readers read that text.

Browsers will display that text if user turns off images

Indexers and search engines use it to know what the image is about
